# Epiplexity 原理 | 认知科学核心概念

本文档详细解释 Epiplexity 原理及其在书籍拆解中的应用。

---

## 核心概念

来自论文《From Entropy to Epiplexity》的核心洞见：

### 传统观点 vs Epiplexity

| 传统观点 | Epiplexity |
|---------|-----------|
| 信息是数据的固有属性 | 信息是相对的，取决于观察者的"认知算力" |
| 书的难度是客观的 | 书的难度取决于你投入的算力 |

### 核心公式

```
同一本书 = 可学习的结构(S) + 不可学习的噪声(N)
```

**关键洞见**：S 和 N 的切割线位置，取决于你投入多少算力。

```
算力低 → 大部分内容看起来是"噪声"
算力高 → 更多内容变成"可学习的结构"
```

---

## 实践含义

1. **读不懂不是因为书太难**
   - 是你投入的算力还不够
   - 通过结构化方法可以提升算力

2. **增加算力的方式**：
   - 多轮阅读（不是简单重复，而是递进深入）
   - 多视角审视（作者视角、批判视角、应用视角）
   - 主动生成（输出倒逼输入）
   - 建立连接（与已有知识关联）

3. **本 Skill 的目标**：
   - 通过"三轮认知压缩"结构化流程
   - 最大化你的"认知算力投入"
   - 将更多内容从"噪声"转化为"可学习的结构"

---

## 应用示例

### 低算力阅读
- 从头读到尾，被动接收
- 只关注字面意思
- 读完就忘，无法复述

### 高算力阅读（Epiplexity 方式）
- 骨架扫描：先建立全局地图
- 血肉解剖：深入理解论证逻辑
- 灵魂提取：与已有知识建立连接
- 输出倒逼：生成笔记/文章/行动清单
