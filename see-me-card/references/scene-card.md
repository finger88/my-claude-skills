# Evidence Scene Card

证据回场卡的目标不是再次解释用户，而是让用户在低精力状态下，凭一张卡重新碰到“想法发生的那一刻”。

它借鉴“飞飞式顿悟瞬间”的结构：先确定人物当时的心境，再构造场域，再让触发原文、用户想法和通感意象在同一瞬间发生。卡片必须有真实证据作锚，通感只负责把心理动作变成可感场，不负责编造事实。

## 生成顺序

1. **证据锚点**
   - 从当日 `evidence.md` 选择 1 条最有触发感的证据。
   - 必须保留：书名、章节、时间、用户原文想法、触发原文/划线。
   - 优先选“用户想法和触发原文之间有明显电位差”的证据：原文说的是 A，用户忽然看见了 B。

2. **心境压缩**
   - 只从证据推断当时的心理张力，不补写没有证据的生活细节。
   - 问三句：
     - 这条想法出现前，用户被什么卡住？
     - 原文中的哪一句让卡住的东西松动？
     - 用户写下想法时，守住了什么？

3. **场域定义**
   - 场域不等于真实房间。若没有生活证据，不写天气、地点、姿势、器物。
   - 场域来自文本关系：书页、问题、答案、工具、时间、声音、光、重量、距离、流动、边界。
   - 用 2-3 个感官维度就够：例如“光变清”“问题有重量”“答案像洪流”“句子推回掌心”。

4. **通感转译**
   - 把抽象心理动作转译为可感动作：
     - 理解 -> 点亮、对焦、穿透、听见
     - 压力 -> 重量、粘滞、窄门、噪声
     - 主体性 -> 握回掌心、站稳、把边界钉住
     - 方法内化 -> 从纸面进入手上、从规则变成步伐
     - 自由 -> 空气重新流动、留出可选择的距离
   - 通感句必须服务证据，不能为了美而飘走。

5. **落笔瞬间**
   - 用户原文想法是“显影发生处”，必须醒目出现。
   - 不要把用户原文改写成更漂亮的话；可以在旁边放一行通感转译，让原文自己发光。

## 卡片结构

推荐字段写法：

- `kicker`: `证据回场`
- `subject`: `微信读书 / <书名> / <日期>`
- `title`: 一个身体化标题，6-12 字，两行以内。例如 `问题回到\n掌心`
- `subtitle`: 时间与心境，不超过 34 个汉字。
- `sourceEcho`: 用户原文想法。
- `mainText`: 触发原文 + 通感转译，36-58 个汉字。
- `subText`: 章节、日期、证据限定，尽量短。
- `nodes`: 3 条左右，建议为：
  - `01 原句`: 触发原文中最锋利的一句。
  - `02 心象`: 从证据推断出的感官场域。
  - `03 落笔`: 用户原文想法如何浮出。
- `closingLead` + `closingStrong`: 不总结人格，只把用户带回那个动作。

## 禁止事项

- 不编造真实环境：不知道用户当时在哪里，就不要写“窗边”“夜色”“咖啡”“键盘声”。
- 不把通感写成鸡汤：不能用“温暖、治愈、拥抱自己”等泛化词。
- 不做二次长分析：回场卡的解释密度必须低于显影卡。
- 不让修辞压过证据：每个意象都要能回指到触发原文或用户原文想法。
- 不堆满整张卡：疲惫时可进入，比信息完整更重要。

## 自检

出图前检查：

- 去掉书名、日期后，这张卡是否仍能让人知道“触发点”在哪里？
- 每个通感意象是否能对应到一条证据，而不是凭空抒情？
- 用户原文想法是否仍是主角？
- 低精力状态下，是否可以先只读标题、主句、原文想法就进入现场？
