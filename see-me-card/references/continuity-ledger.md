# Continuity Ledger

每日看见不是断裂的卡片，而是一条持续显影的河流。使用连续台账保存“可修正的工作假设”，让新 session 也能承接旧水流。

## 台账位置

默认位置：

```text
D:\my tool\memory-work\02 你的阅读\每日看见我\see-me-ledger.md
```

这个文件不放在 `关于我/` 下，避免把阶段性显影误写成高敏感长期档案。它是实验台账，不是定论。

如果目录不存在，可以创建：

```text
D:\my tool\memory-work\02 你的阅读\每日看见我\
```

## 读取规则

每次运行前：

1. 如果台账存在，读取最近 7-14 条日卡记录。
2. 读取 `## 开放假设` 和 `## 反复出现的线索`。
3. 把台账作为 `ledger prior`，只能辅助主题选择，不能替代当日微信读书证据。
4. 如果当日证据反驳台账中的假设，优先相信当日证据，并记录“修正”。

## 写入规则

每次卡片生成并验图后，追加一条记录：

```markdown
## YYYY-MM-DD 主题词

- 卡片：
- 数据源：
- 数据范围：
- sourceEcho：
- 今日主文眼：
- 新沉积：
- 延续了哪些旧线索：
- 修正了哪些旧判断：
- 证据水位：
- 下一次可追：
```

再更新两个区块：

```markdown
## 反复出现的线索

- 线索：证据摘要；出现日期；稳定度（低/中/高）

## 开放假设

- 假设：支持证据；反证/缺口；下一步需要的数据
```

## 写法原则

- 用“可能”“今天显示”“阶段性”而不是“你就是”。
- 不写人格判词，不写医学/心理诊断。
- 不把单日情绪当长期结构。
- 不把台账写进 `MEMORY.md`，除非用户明确要求并逐条确认。
- 台账可被后来证据推翻；推翻不是失败，是显影变清楚。

## 证据水位

- `低`：单条想法或单日材料。
- `中`：同一本书 2-5 条想法，跨章节或跨日期。
- `高`：多本书/多来源/跨时间段共同指向。

## 交付时说明

最终回复要说明：

- 今天用了哪些台账 prior。
- 今天新增了什么沉积。
- 哪个旧判断被加强、分化或修正。
