### 1. Windows路径错误 - Python无法打开文件

| 维度 | 内容 |
|------|------|
| **现象** | Python报错 `can't open file`，路径显示为 `D:\c\Users\...` 格式，明显路径解析错误 |
| **根因** | Windows环境下路径分隔符 `\` 被转义或路径展开时 `~` 符号处理错误，导致路径拼接异常 |
| **解决方案** | 使用绝对路径，或将 `\` 替换为 `/`，或使用 `pathlib.Path` 处理跨平台路径。避免在Windows下使用 `~` 展开为用户目录 |
| **检测信号** | 看到 `can't open file` + 路径中有 `\c\` 或其他异常片段 |

*记录时间: 2026-03-25*


### 2. WebFetch安全限制无法抓取特定域名

| 维度 | 内容 |
|------|------|
| **现象** | 使用WebFetch工具抓取微信文章等特定域名时返回空内容或安全警告，无法获取页面内容 |
| **根因** | WebFetch工具有安全沙箱限制，对微信域名(weixin.qq.com等)设置了访问限制，防止恶意抓取 |
| **解决方案** | 对于受限制域名，改用web-access MCP工具或手动复制内容。对于微信文章，可先用浏览器打开再复制正文 |
| **检测信号** | WebFetch返回空内容或 `access denied` / `security restriction` 提示 |

*记录时间: 2026-03-25*


### 3. jina.ai触发CAPTCHA返回验证页面

| 维度 | 内容 |
|------|------|
| **现象** | 通过jina.ai API提取文章内容时，返回的是CAPTCHA验证页面HTML而非正文内容 |
| **根因** | 目标网站检测到非人类访问流量，触发反爬机制要求验证。jina.ai作为代理服务无法自动处理CAPTCHA |
| **解决方案** | 1) 减少请求频率，添加延时 2) 更换User-Agent 3) 对于CAPTCHA强制网站，改用人工复制或专用API 4) 考虑使用付费内容提取服务 |
| **检测信号** | 返回内容包含 `captcha`、`验证`、`security check` 等关键词，或明显是HTML页面而非正文 |

*记录时间: 2026-03-25*

## 最佳实践

### DO（应该做的）
- [x] 使用 `pathlib.Path` 处理跨平台路径问题
- [x] 遇到WebFetch限制时，及时切换替代方案
- [x] 对于高频请求，添加延时和重试机制
- [x] 保存原始错误信息便于后续分析

### DON'T（避免做的）
- ❌ 在Windows下直接使用 `~` 作为用户目录缩写
- ❌ 对受保护域名（如微信）强行使用WebFetch
- ❌ 忽略CAPTCHA信号继续重试（可能触发封禁）
- ❌ 混用 `\\` 和 `/` 作为路径分隔符
