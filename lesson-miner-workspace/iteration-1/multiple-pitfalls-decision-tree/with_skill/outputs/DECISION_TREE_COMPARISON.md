## Changes Summary

1. **新增分支**: 连接池满检测（第4分支）
2. **结构变化**: 从3层决策扩展到4层
3. **检测信号**: 基于"看到 Connection pool exhausted"快速识别
4. **最佳实践更新**: 新增连接池监控和配置建议

## 新增卡点详情

**标题**: 数据库连接池满导致请求超时

**现象**: 应用请求超时，日志显示 'Connection pool exhausted' 或 'Timeout waiting for connection'

**根因**: 数据库连接池配置过小，或连接未及时释放，或连接泄漏导致池满

**解决方案**: 1. 增加连接池最大连接数（如 HikariCP 调整 maximum-pool-size）<br>2. 检查连接泄漏，确保 try-with-resources 关闭连接<br>3. 设置合理的连接超时时间<br>4. 监控连接池使用率

**检测信号**: 看到 'Connection pool exhausted' 或大量请求超时 + 数据库服务正常
