## 卡点记录

### 1. 数据库防火墙阻止连接

| 维度 | 内容 |
|------|------|
| **现象** | 连接数据库时报错 'Connection timed out'，无法建立连接 |
| **根因** | 数据库服务器防火墙未开放 3306 端口，或安全组规则限制 |
| **解决方案** | 1. 检查服务器防火墙设置，开放 3306 端口<br>2. 检查云平台安全组规则<br>3. 确认 IP 白名单包含应用服务器 IP |
| **检测信号** | 看到 'Connection timed out' + 网络可达但端口不通 |

*记录时间: 2025-03-20*

### 2. 数据库连接字符串格式错误

| 维度 | 内容 |
|------|------|
| **现象** | 报错 'Access denied for user' 或 'Unknown database' |
| **根因** | 连接字符串中用户名、密码或数据库名拼写错误，或特殊字符未转义 |
| **解决方案** | 1. 核对用户名/密码拼写<br>2. 对特殊字符进行 URL 编码<br>3. 使用配置文件分离敏感信息 |
| **检测信号** | 看到认证相关错误 + 确认网络连接正常 |

*记录时间: 2025-03-20*

### 3. 环境变量未加载导致配置缺失

| 维度 | 内容 |
|------|------|
| **现象** | 应用启动时报 'DATABASE_URL is not set' 或配置项为空 |
| **根因** | .env 文件未加载，或环境变量未正确导出，或配置文件路径错误 |
| **解决方案** | 1. 确认 .env 文件存在且格式正确<br>2. 使用 dotenv 库加载环境变量<br>3. 检查文件权限，确保可读 |
| **检测信号** | 看到配置项缺失错误 + 应用刚部署或刚重启 |

*记录时间: 2025-03-20*

### 4. 数据库连接池满导致请求超时

| 维度 | 内容 |
|------|------|
| **现象** | 应用请求超时，日志显示 'Connection pool exhausted' 或 'Timeout waiting for connection' |
| **根因** | 数据库连接池配置过小，或连接未及时释放，或连接泄漏导致池满 |
| **解决方案** | 1. 增加连接池最大连接数（如 HikariCP 调整 maximum-pool-size）<br>2. 检查连接泄漏，确保 try-with-resources 关闭连接<br>3. 设置合理的连接超时时间<br>4. 监控连接池使用率 |
| **检测信号** | 看到 'Connection pool exhausted' 或大量请求超时 + 数据库服务正常 |

*记录时间: 2025-03-25*

## 最佳实践

### DO
- [ ] 使用连接池管理数据库连接（推荐 HikariCP）
- [ ] 配置连接超时和重试机制
- [ ] 将数据库配置与代码分离
- [ ] 使用环境变量存储敏感信息
- [ ] 监控连接池使用率（目标 < 80%）
- [ ] 使用 try-with-resources 确保连接关闭

### DON'T
- ❌ 在代码中硬编码数据库密码
- ❌ 使用 root 账户连接应用数据库
- ❌ 忽略连接错误处理
- ❌ 无限制地创建新连接而不关闭旧连接
- ❌ 连接池配置过大（浪费资源）或过小（导致超时）
- ❌ 忽略连接池监控和告警
